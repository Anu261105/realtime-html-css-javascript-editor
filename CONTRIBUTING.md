# Contributing to LiveForge

Thank you for your interest in contributing to LiveForge.

LiveForge is an open-source real-time HTML, CSS and JavaScript editor with instant live preview, syntax highlighting, local autosave and an integrated browser console.

Contributions of all sizes are welcome, including bug fixes, documentation improvements, accessibility enhancements, UI refinements and new features.

## Ways to Contribute

You can contribute by:

- Reporting bugs
- Suggesting new features
- Improving documentation
- Fixing UI or responsive design issues
- Improving accessibility
- Enhancing editor functionality
- Improving performance
- Submitting code fixes
- Reviewing open issues and pull requests

## Getting Started

### 1. Fork the Repository

Fork the repository to your own GitHub account.

### 2. Clone Your Fork

```bash
git clone https://github.com/YOUR_USERNAME/realtime-html-css-javascript-editor.git
```

### 3. Open the Project Directory

```bash
cd realtime-html-css-javascript-editor
```

### 4. Create a Feature Branch

Use a descriptive branch name:

```bash
git checkout -b feature/device-preview
```

Other examples:

```text
fix/console-error-handling
docs/improve-readme
feature/project-management
refactor/editor-layout
```

### 5. Run the Project Locally

Open `index.html` directly in a modern browser or use the Live Server extension in VS Code.

No npm installation, backend server or build process is required.

## Development Guidelines

When contributing:

- Keep changes focused and easy to review
- Follow the existing project structure
- Use clear and descriptive names
- Avoid unnecessary dependencies
- Test changes in a modern browser
- Preserve responsive behavior
- Keep accessibility in mind
- Do not commit secrets, API keys or credentials
- Update documentation when behavior changes

## Commit Message Style

Use clear commit messages. Conventional-style prefixes are encouraged:

```text
feat: add responsive device preview
fix: resolve console rendering issue
docs: improve setup instructions
style: refine editor toolbar spacing
refactor: simplify preview update logic
chore: update repository configuration
```

## Pull Request Process

Before opening a pull request:

1. Test your changes locally
2. Check that existing functionality still works
3. Keep the pull request focused on one main improvement
4. Describe what changed and why
5. Include screenshots for visible UI changes
6. Link related issues when applicable

## Reporting Bugs

When reporting a bug, include:

- A clear title
- Steps to reproduce the issue
- Expected behavior
- Actual behavior
- Browser and operating system
- Screenshots or console errors when useful

## Suggesting Features

Feature suggestions are welcome.

Please explain:

- The problem the feature would solve
- The proposed behavior
- Why it would be useful
- Any possible alternatives

## Code of Conduct

Be respectful, constructive and professional when participating in project discussions.

## Questions

For questions about the project, open a GitHub Discussion or Issue with enough context for others to understand the request.

Thank you for helping improve LiveForge.
